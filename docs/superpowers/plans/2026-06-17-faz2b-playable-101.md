### Task 2: `@cs-okey/bot` 101 decision

**Files:** Modify `packages\bot\src\index.ts`; Test extend `bot.test.ts`.
**Interfaces:** `decide(view, legal, rng)` — when `view.config.requiresOpening` (101):
- DRAW phase: same as Klasik (take left if useful else stock).
- DISCARD phase: if `legal` includes `DeclareWin` and a winning discard exists AND `view.you.hasOpened` → DeclareWin. Else if NOT opened and `findOpening(view.you.rack, view.okey, view.config)` returns a set → return `OpenMeld` with that set. Else if opened → try a `LayOff` (find a rack tile that extends a `view.tableMelds` meld; if found return LayOff) else Discard least-useful. Else (can't open) → Discard least-useful.
- Never returns an illegal move type (guard against `legal`). Keep Klasik path unchanged.

- [ ] Step 1: Tests — a 101 view where the rack can open → decide returns OpenMeld; a 101 view, opened + winning → DeclareWin; a 101 view, can't open → Discard. (Construct PlayerViews with config=KLASIK_101.)
- [ ] Steps 2-5: Implement, run, PASS, full suite green, lint, commit `feat(bot): 101 decision (open ≥101 / layoff / declare / discard)`.
