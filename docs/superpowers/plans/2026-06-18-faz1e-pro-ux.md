# Faz 1e — Drag-to-Discard + Auto-Finish + Pro Chrome Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Checkbox steps. VISUAL/INTERACTIVE — controller verifies in the browser (screenshots) at each task.

**Goal:** Make discarding work by **drag-to-discard** (drag a rack tile onto the discard zone to your right), **auto-detect finishing** (no separate "Bitir" button — when your final discard empties the rack / completes the hand, you win automatically), remove the illogical "Taş At"/"Elimi Aç-Bitir" buttons, and give the board **professional chrome**: proper nameplates for all 4 players (including the human, currently missing), refined smaller buttons, a polished bottom-centre rack, and correctly-placed discard drop zones.

**Architecture:** Lift the dnd-kit `DndContext` from inside `SlotRack` up to `GameScreen` so it wraps BOTH the rack slots AND a discard drop-zone; on drag-end: drop on a slot → rearrange (moveTile), drop on the discard zone → dispatch `Discard`. The engine's `Discard` reducer gains **auto-finish**: after removing the discarded tile, if the hand is complete (Klasik: remaining 14 form a winning arrangement; 101: rack empty and opened) it ends as a win — so the discard IS the finishing move.

**Tech Stack:** Same. dnd-kit. Browser verification by controller.

### Task 1: Engine — auto-finish on Discard (both variants)

**Files:** `reduce.ts`, `rules/klasik.ts`, `rules/yuzbir.ts`; Test `autoFinish.test.ts`.
**Interfaces:** `Discard` reducer: after removing the discarded tile from the player's rack:
- Klasik (`!config.requiresOpening`): `const r = evaluateHand(rack, okey, config)`; if `r.isWinning` → `status:'ENDED'`, `terminal:{reason:'win', winnerSeat:seat, winType:r.winKind, finishingTile:discardedTile}`, mark player isOut, push the discard. Else normal (advance turn).
- 101 (`config.requiresOpening`): if `rack.length === 0 && player.hasOpened` → terminal win (winType: 'perOnly' or from context; finishingTile = discarded). Else normal.
- `legalMoves`/`legalMoves101`: DISCARD phase still offers `Discard` (now the finisher); `DeclareWin` may stay listed but is no longer required by the UI.
